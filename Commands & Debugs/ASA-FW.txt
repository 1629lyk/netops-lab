# SW2-L3 ping 8.8.8.8

Requests and replies are both passing through R1 perfectly. R1 is forwarding correctly in both directions. The problem is the ASA is dropping the return traffic before it reaches SW2-L3.
The replies arrive at R1 eth2, R1 forwards them to ASA outside (10.0.13.2), but ASA's stateful inspection is dropping them because ASA didn't see the original outbound session initiated from inside — or the return traffic is hitting the outside interface and ASA's security model blocks outside-initiated traffic by default.
Fix on ASA — enable stateful inspection to allow return traffic:
configure terminal

! Allow established/related return traffic on outside interface
access-list OUTSIDE_IN extended permit icmp any any
access-group OUTSIDE_IN in interface outside

! Also ensure inspection policy is active
policy-map global_policy
 class inspection_default
  inspect icmp
service-policy global_policy global

write memory




Add static routes on ASA for the VLAN subnets:
configure terminal
route inside 10.10.0.0 255.255.255.0 10.0.14.2
route inside 10.20.0.0 255.255.255.0 10.0.14.2
write memory
